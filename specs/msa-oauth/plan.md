# Implementation Plan: MSA OAuth Authentication

**Branch**: `feat/msa-oauth` | **Date**: 2026-03-24 | **Spec**: user request
**Input**: MCP server should authenticate on behalf of a specific user and obtain the associated MS Graph API permissions.

## Summary

Add OAuth 2.0 authorization code flow with PKCE to the MCP server so it can authenticate a Microsoft personal account (MSA) user and obtain access tokens for calling the Microsoft Graph API. Uses MSAL for auth, stores tokens server-side, and enforces the existing user allowlist.

## Technical Context

**Language/Version**: Python 3.13
**Primary Dependencies**: FastMCP, Starlette, MSAL, httpx
**Storage**: MSAL serialized token cache (file-backed, server-side)
**Testing**: pytest
**Target Platform**: Linux server (Azure App Service)
**Project Type**: web-service (MCP server)
**Performance Goals**: N/A (single-user, low-throughput auth flow)
**Constraints**: Consumer MSA accounts only, HTTPS required, tokens never exposed to AI client
**Scale/Scope**: Single allowed user (expandable via env var)
